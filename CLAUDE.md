# EASCO Content Repository

**Content-only repo. No code, HTML, CSS, or frameworks. Decline any build request.**

## Start of Every Session — Ask First
> **"Agent mode or manual mode?"**
> - **Agent mode** — read `_system/memory.md` + `_system/execution-plan.md`, complete the first unchecked task through all five passes, update `_system/progress.md` + `_system/memory.md`, commit, push, stop.
> - **Manual mode** — read `_system/memory.md`, then wait for user instructions.

## Always Read Before Writing Any Page
`_system/WritingSystem.md` (full — includes Data Source Hierarchy, Verbatim Authority Statements, Pass N Term Lists) · `_system/VERIFIED-COMPANY-DATA.md` · the stub's frontmatter (= page brief) · the template it names.
Templates: category/archive → `templates/product-archive-template.md` · product detail → `templates/product-page-template.md` · others per stub frontmatter. Slugs: `_system/sitemap.md`. Design specs: `_system/design.md` (never in copy files).

## Definition of Done
Five passes (A→B→N→D→C) complete · every number traced per Data Source Hierarchy · no banned claims · min 2 differentiators · pass log header added · `_system/progress.md` + `_system/memory.md` updated · committed and pushed.

## Memory
`_system/memory.md` holds current state, standing decisions, open questions, session log. Update it at the end of every session — it is the handover to the next session.

## Git
One page per commit: `[WRITE] path/file.md — five-pass complete`. System changes: `[SYSTEM]` prefix.
