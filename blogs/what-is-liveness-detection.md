### SECTION 1: Intro / Hook

**H1: What Is Liveness Detection?**

Liveness detection is the part of a biometric system that confirms a real, live person is present during a scan. It rules out a photo, a video replay, or a mask held up to the camera. ### SECTION 2: How Liveness Detection Actually Works

**H2: Telling a Live Face from a Flat Image**

A face recognition camera alone only compares an image against an enrolled template. It doesn't inherently know whether that image came from a live person or a printed photo held up to the lens. Liveness detection adds a second layer of analysis specifically to answer that question. Depending on the system, it can check for subtle depth cues a flat photo lacks, or natural micro-movements a static image can't replicate. Some systems also check infrared reflectance, which differs between skin and paper or screen surfaces.

The quality tier matters enormously here. A basic system might only check for blinking, which a short video loop can fake. A financial-grade system layers multiple checks instead, depth, texture, infrared response, and movement. That makes it far harder to spoof with a photo, video, or even a realistic silicone mask.

> Liveness detection confirms a live person is present during a biometric scan, using depth analysis, infrared reflectance, or micro-movement detection to distinguish a real face from a photo, video, or mask. Financial-grade liveness detection layers multiple checks; basic consumer-grade systems often rely on a single, easier-to-fake signal like blinking.

### SECTION 3: Four Questions to Ask About Any Liveness Detection Claim

**H2: "Anti-Spoofing" Isn't a Single Standard**

1. **What specific spoofing methods does it defend against?** Photos, video replays, and 3D masks each require different detection methods. Ask which ones are covered, not just whether "anti-spoofing" is included.
2. **Is it single-factor or multi-factor?** A system checking only one signal (like blinking) is generally easier to fool than one layering depth, infrared, and movement analysis together.
3. **What's the false rejection impact?** Stricter liveness detection can sometimes increase false rejections for legitimate users. Ask how the system balances security against convenience.
4. **Has it been tested against realistic spoofing attempts?** A vendor should be able to describe what testing standard or methodology their liveness detection has been evaluated against.

### SECTION 6: FAQ

**H2: Frequently Asked Questions**

**Q: What is liveness detection in face recognition?**
A: A layer of analysis that confirms a real, live person is present during a scan, distinguishing them from a photo, video replay, or mask.

**Q: Can liveness detection be fooled?**
A: Basic single-factor systems (like blink detection alone) are easier to fool than financial-grade systems layering depth, infrared, and movement analysis together.

**Q: Does liveness detection slow down the scan?**
A: Well-designed systems add negligible time, EASCO's face recognition terminals complete liveness detection within the same 200–300ms overall scan window.

**Q: Is liveness detection the same as anti-spoofing?**
A: Yes, the terms are generally used interchangeably to describe technology that distinguishes a live person from a fake presentation.

**Q: Do all face recognition systems include liveness detection?**
A: No. Confirm specifically whether liveness detection is included and at what quality tier, some basic systems skip it entirely or use minimal checks.
