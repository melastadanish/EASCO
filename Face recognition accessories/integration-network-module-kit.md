### SECTION 5 — Feature Deep-Dive Blocks

**H2: Three Integration Standards on One Platform — Choose What You Need**

The module ships with Wiegand 26/34 connectors as standard — the most common protocol used in older access control systems and most lock controllers. Wiegand is simple TTL-level signaling that operates up to 100 meters over standard twisted pair cable. For longer distances or noisy electrical environments, the RS485 differential interface is included and accepts external terminal blocks for field termination. Modern building management systems and networked access panels use Ethernet (TCP/IP), which the module supports via an RJ45 connector and built-in protocol translation. All three protocols can run simultaneously on the same biometric terminal — meaning an integrator can wire Wiegand to a legacy lock, RS485 to a building panel, and Ethernet to a central management system, all active at the same time.

**H2: Surge Protection and Isolated Relay Outputs — Protect Against Field Hazards**

Every signal line entering or exiting the module is protected by 2kV transient surge suppressors that absorb electrical spikes from door locks, motor starters, and lightning strikes before they reach the biometric terminal's internal circuits. Relay outputs provide isolated switching for low-voltage control circuits (24V DC gate openers, door mag locks, alarm outputs). Because relays use electromagnetic switching rather than solid-state circuits, they do not conduct electrical noise back into the biometric terminal — critical in industrial environments with motor-driven gates or high-current solenoid locks. The module mounts on a standard 15mm DIN rail using rail-mounted terminal blocks, making installation consistent with electrical code across all facilities.

### SECTION 12 — FAQ

**H2: Frequently Asked Questions**

**Q: Can I use all three protocols (Wiegand, RS485, Ethernet) at the same time?**
A: Yes. The Integration Module is designed to support simultaneous operation of all three protocols on a single biometric terminal. You can route Wiegand output to a legacy lock, RS485 to a building panel, and Ethernet to a management system — all active together. This flexibility eliminates the need for separate terminals for each protocol.

**Q: What's the difference between Wiegand, RS485, and Ethernet protocols?**
A: Wiegand is a simple point-to-point interface used by older door lock controllers — it sends 26 or 34 bits of data when a face is recognized. RS485 is a differential serial protocol for longer distances (up to 1200 meters) and noisier electrical environments — used by many building access panels. Ethernet (TCP/IP) is modern networked protocol for real-time data exchange with building management systems. Integrators choose based on their existing access control infrastructure.

**Q: Do I need special firmware to use the Integration Module?**
A: No. The EASCO biometric terminal recognizes the module automatically via its internal protocol detection. On first power-up, the terminal identifies which protocols are connected and automatically enables the appropriate interface drivers. No configuration files or firmware updates required for standard installations.

**Q: How do I protect the module against electrical surges from door locks and motor-driven gates?**
A: The module includes 2kV transient surge suppressors on every signal line. These absorb high-voltage spikes (from motor starters, lightning strikes, and solenoid coils) before they reach the terminal's internal circuits. Surge protection is always active — no configuration required.

**Q: What happens if a relay output needs to control a high-current device (door opener with large solenoid)?**
A: The relay output is rated 24V DC / 2A per contact. For higher currents, use the relay output to trigger an external contactor or solid-state relay that handles the higher current. This is standard electrical practice and isolates the biometric terminal from high-current switching noise.

**Q: Can the module be used with non-EASCO biometric terminals?**
A: The module is optimized for EASCO face recognition terminals with compatible connector types. For third-party terminals, confirm that they support Wiegand 26/34, RS485, or Ethernet protocols and have compatible signal voltage levels (TTL-level for Wiegand, 3.3V/5V for logic). EASCO can customize connector types upon request.
