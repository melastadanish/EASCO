## Writing Instructions

### ⚠️ CLIENT DATA REQUIRED

**CRITICAL:** Case study pages CANNOT be written until client provides:
- Project names and locations (city, country)
- Products used (model numbers)
- Challenge / requirement the client had
- Solution EASCO provided
- Result / outcome (measurable if possible)
- **Installation photos** (minimum 3–5 per project)

If client cannot provide:
- Write Sections 6–10 (requirements, products, FAQ, CTA, related)
- Use placeholder text in Sections 2–5 with clear `[CLIENT DATA NEEDED]` markers
- Focus on venue-type requirements vs. specific projects

### Tier 1 Checks (Run after EVERY section)
1. **Spec-backed:** Product references traced to client-data-map.md
2. **Objection coverage:** Address venue-specific objections
3. **Social proof:** Real projects with locations build trust
4. **Keyword density:** Target keyword 3–5 times naturally
5. **CTA clarity:** Clear contact path for similar projects

### Target Length
- Total: 1,500–2,200 words (depending on number of case studies)
- Section 1 (Hero): 100–150 words
- Section 2–4 (Case Studies): 400–500 words each
- Section 5 (Other Projects): 100–150 words
- Section 6 (Requirements): 300–400 words
- Section 7 (Products): 200–250 words
- Section 8 (FAQ): 400–500 words
- Section 9 (CTA): 100 words
- Section 10 (Related): 50 words

### Five-Pass Review (After full page written)

#### Pass A: Full-Page Quality Audit
Score each dimension 1–10:
1. **Credibility** — real projects with locations/photos
2. **Specificity** — actual product models and configurations
3. **Relevance** — venue-type requirements covered
4. **EASCO differentiation** — why EASCO vs alternatives
5. **Keyword optimization** — natural SEO
6. **Conversion clarity** — clear path to contact

**Minimum: 8.0/10 overall**

#### Pass B: Surgical Rewrite
Fix ONLY sections that scored <8 in Pass A.

#### Pass N: NLP Semantic Coverage
Required terms:
- "case study" or "project"
- [venue type] (e.g., office building, school, airport)
- At least 2 product terms (turnstile, face recognition, etc.)
- Location terms (city/country)
- Result terms (accuracy, throughput, etc.)

#### Pass D: E-E-A-T / GEO Check
- [ ] One AI-quotable paragraph (project outcome with measurable result if available)
- [ ] FAQ with 5+ Q&As specific to venue type
- [ ] Authority signals: real project locations, integration details
- [ ] Freshness: "Project completed [date]" or "Updated [month year]"

#### Pass C: Conversion Review
- [ ] Test 1: CTA present after each case study ✓
- [ ] Test 2: WhatsApp contact visible ✓
- [ ] Test 3: Social proof elements (locations, results) ✓
- [ ] Test 4: Product links present ✓
- [ ] Test 5: Clear "get similar solution" CTA at end ✓
- [ ] Test 6: AI-written score ≤ 15%
